# Factory Pulse

스마트 팩토리 운영자를 위한 공장, 생산 라인, 설비, 알람, 생산 리포트 모니터링 서비스

Factory Pulse는 제조 현장의 운영 상태를 한 화면에서 빠르게 확인하기 위한 내부 운영 대시보드입니다.

공장별 생산 현황, 라인별 가동 상태, 설비 이상 신호, 알람 우선순위, 생산 리포트를 분리된 화면에서 확인하고, 운영자가 지금 어디를 먼저 봐야 하는지 판단할 수 있도록 돕습니다.

## 서비스 소개

Factory Pulse는 MES, ERP, SCADA처럼 거대한 현장 시스템을 대체하려는 서비스가 아닙니다.

이미 존재하는 생산 설비와 운영 데이터를 Supabase 기반 데이터베이스로 정리하고, 현장 관리자나 개발자가 웹 대시보드에서 공장 상태를 빠르게 확인할 수 있도록 만든 스마트 팩토리 모니터링 프로젝트입니다.

초기 MVP는 실제 제어 명령보다 **조회, 상태 판단, 알람 확인, 생산 흐름 파악**에 집중합니다.

## 이런 사용자에게 맞습니다

- 여러 공장의 생산 상태를 한 화면에서 확인해야 하는 운영 관리자
- 라인별 가동률, 생산량, 품질 지표를 빠르게 비교해야 하는 생산 관리자
- 설비의 온도, 진동, 전력, 상태 알람을 지속적으로 확인해야 하는 설비 담당자
- 알람의 심각도와 발생 위치를 기준으로 대응 순서를 정해야 하는 현장 관리자
- 스마트 팩토리 데이터를 웹 서비스 구조로 설계하고 싶은 풀스택 개발자

## 해결하려는 문제

제조 현장 데이터는 공장, 라인, 설비, 센서, 알람, 일일 리포트처럼 서로 다른 단위로 흩어져 있습니다.

Factory Pulse는 이 데이터를 운영자가 이해하기 쉬운 화면 흐름으로 묶습니다.

```text
공장 상태 확인
  -> 위험 라인과 설비 파악
  -> 알람 심각도 확인
  -> 설비 상세에서 현재 센서 상태 확인
  -> 생산 리포트로 기간별 흐름 확인
```

## 핵심 가치

### 현장 상태를 빠르게 스캔

대시보드에서 전체 KPI, 위험 설비, 최근 알람, 생산 추이를 먼저 보여줍니다.

운영자는 모든 데이터를 하나씩 찾기보다 현재 문제가 있는 영역을 먼저 확인할 수 있습니다.

### 데이터 최신성에 맞는 화면 전략

자주 바뀌지 않는 공장 정보는 ISR로 제공하고, 실시간성이 중요한 대시보드와 설비 상세는 Dynamic Rendering으로 처리합니다.

사용자 입력으로 바뀌는 검색, 필터, 비교 화면은 CSR로 분리해 화면 반응성을 높입니다.

### 알람 중심의 운영 흐름

알람은 단순 목록이 아니라 공장, 라인, 설비, 심각도, 해결 여부와 연결됩니다.

운영자는 알람을 보고 바로 관련 설비 상세, 공장 상세, 리포트 화면으로 이동할 수 있습니다.

### Supabase 기반 확장성

공장, 라인, 설비, 센서 데이터, 알람, 생산 리포트를 Supabase Postgres에 저장합니다.

RLS 정책을 기준으로 공개 조회와 인증이 필요한 변경 작업을 분리할 수 있도록 설계했습니다.

## 현재 구현된 화면

### 홈

- Factory Pulse 프로젝트 소개
- 주요 화면으로 이동하는 진입점
- 대시보드, 공장, 설비, 알람, 리포트 링크 제공

### 대시보드

- 전체 공장 수, 설비 수, 활성 알람 수, 평균 가동률 확인
- 위험 설비 보드 확인
- 최근 알람 확인
- 생산 추이 확인
- 공장, 설비, 알람, 리포트 화면으로 이동

### 공장 관리

- 공장 목록 조회
- 지역, 상태, 검색 필터
- 공장별 라인 수, 설비 수, 활성 알람 수 확인
- 공장 상세 화면 확인
- 공장 상세에서 해당 공장의 설비/리포트 화면으로 이동

### 설비 모니터링

- 설비 목록 조회
- 공장, 상태, 검색 필터
- 설비별 상태, 가동률, 생산량, 불량률 확인
- 설비 상세에서 현재 센서 스냅샷 확인
- 관련 최근 알람 확인
- 설비 기준 알람 목록으로 이동

### 알람 관리

- 전체 알람 목록 조회
- 심각도, 해결 여부, 공장, 설비 기준 필터
- 알람별 발생 위치와 메시지 확인
- 관련 설비 상세로 이동

### 생산 리포트

- 일별 생산 리포트 조회
- 공장별 생산량, 목표 달성률, 가동률, 품질 지표 확인
- 기간/공장 기준 필터
- 생산 흐름 차트 확인

## MVP 범위

현재 MVP는 스마트 팩토리 운영자가 데이터를 조회하고 상태를 판단하는 흐름에 집중합니다.

포함하는 범위:

- 홈, 대시보드, 공장, 설비, 알람, 리포트 화면
- Supabase 기반 공장/라인/설비/센서/알람/리포트 데이터 조회
- 페이지별 렌더링 전략 분리
- 검색, 필터, 상세 이동
- Jest unit test와 Playwright headed Chrome E2E 테스트

제외하는 범위:

- 실제 설비 제어 명령
- MES/ERP/SCADA 실시간 연동
- 작업자 계정별 권한 화면
- 알람 해결 mutation
- 유지보수 작업 등록
- 실시간 WebSocket 알람 수신
- 배포 환경의 인증/인가 플로우

## 제품 포지션

Factory Pulse는 현장 설비를 직접 제어하는 시스템이 아니라, 스마트 팩토리 데이터를 웹 서비스 형태로 정리해 보여주는 운영 모니터링 도구입니다.

복잡한 제조 시스템 전체를 한 번에 구현하기보다, 풀스택 관점에서 데이터 모델링, Supabase RLS, Next.js 렌더링 전략, 대시보드 UX를 실제 프로젝트처럼 설계하는 데 초점을 둡니다.

## 실행 방법

의존성을 설치합니다.

```bash
npm install
```

`.env.local` 파일에 Supabase 값을 입력합니다.

```bash
NEXT_PUBLIC_SUPABASE_URL=https://프로젝트-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=Supabase publishable 또는 anon key
```

개발 서버를 실행합니다.

```bash
npm run dev
```

3001 포트로 실행하려면 아래 명령을 사용합니다.

```bash
npm run dev -- --port 3001
```
